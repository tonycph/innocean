Introduction of INNOMATI's API documentation

INNOMATI API is powerful tool to control your device with no range limitation. Two steps are required to connect your device. First step is to get the hosts for live streaming and real-time controlling. It can either be obtained by HTTP request or in Device Panel. Second step is to connect and control your device by websocket. If you have any questions, please send us an email at tech@innomati.com.

First Step: Get Host

Endpoint for HTTP request is https://innomati.com/API.php. Below is the request format. Noted that server reads and returns JSON string.

Request

Parameter Type # Description
action String Required Action to execute (See below)
data Array Required Support data for the action (See below)

Sample Request & Response (Javascript)

var xhr = new XMLHttpRequest();
xhr.onreadystatechange = function() {
    if (xhr.readyState == XMLHttpRequest.DONE && xhr.status === 200) {
        response = JSON.parse(xhr.responseText);
        ...
    }
}
xhr.open("POST", ENDPOINT, true);
xhr.setRequestHeader('Content-Type', 'application/json');
xhr.send(JSON.stringify({action: "ACTION", data: {...}}));

Explanation of Response

Key Value
success Boolean - Tells whether our server accepts the request
error Array - Shows error code and message if success is False
data Array - Shows corresponding data if success is True

action: get_device_host

This action is used to obtain device's hosts for live streaming and real-time controlling.

Parameter Type # Description
device_id String Required An unique ID of the device
device_api_key String Required An key assoicated with the device

Response

{
    "success": true,
    "data": {
        "camera_host": {
            "hostname": "3G9E.camera.us.innomati.com",
            "port": "8080"
        },
        "control_host": {
            "hostname": "KR4D.control.us.innomati.com",
            "port": "8443"
        },
    }
}

Explanation of data parameter

Key Value
camera_host Array - Contains camera websocket data of hostname and port
control_host Array - Contains control websocket data of hostname and port

Second Step: Live Streaming & Real-Time Controlling

Connect the hosts using websockets for live streaming (camera_host) and real-time controlling (control_host). Below is the request format.

Live Streaming - The camera host transfer data in h.264 format.

Sample Connection (Javascript)

// https://github.com/131/h264-live-player

var cameraCanvas = document.getElementById("cameraCanvas")
var wsavc = new WSAvcPlayer(cameraCanvas, "webgl");
var url = "wss://" + CAMERA_HOST["hostname"];
url = CAMERA_HOST["port"] == null ? url : url + ":" + CAMERA_HOST["port"];
wsavc.connect(url + "/");

Real-Time Controlling - The device reads and returns JSON string.

Request

Parameter Type # Description
action String Required Action to execute (See below)
data Array Required Support data for the action (See below)

Sample Request & Response (Javascript)

var url = "wss://" + CONTROL_HOST["hostname"];
url = CONTROL_HOST["port"] == null ? url : url + ":" + CONTROL_HOST["port"];
websocket = new WebSocket(url);
websocket.onopen = function (msg) {
    websocket.send(JSON.stringify({"action": ACTION, "data": DATA}));
};
websocket.onmessage = function (event) {
    var request = JSON.parse(event.data);
    var response = request["response"];
    var data = response["data"];
    ...
};

Explanation of Response

Key Value
action String - Same as the request's action
data Array - Same as the request's data
response Array - Response according to request

Explanation of response parameter

Key Value
success Boolean - Tells whether our server accepts the request
error Array - Shows error code and message if success is False
data Array - Shows corresponding data if success is True

action: control

This action is used to control the device manually.

Parameter Type # Description
angle Double Required A number from -1 to 1 with two decimal places
power Double Required A number from -1 to 1 with two decimal places

Response

{
    "success": true
}

action: control_auto

This action is used to control the device automatically with assigned coordinates.

Parameter Type # Description
on String Required Start / stop the automation
coordinates Array Required if on is True A list with coordinates in following format - [(Latitude, Longitude), (Latitude, Longitude), ...]

Response

{
    "success": true
}

action: power_angle_setting

This action is used to control the device automatically with assigned coordinates.

Parameter Type # Description
min_power Double Required A number from 0 to 0.99 to set minimum power
max_power Double Required A number from 0.01 to 1 to set maximum power
calibration_angle Integer Required A number from -45 to 45 to calibrate straight angle
is_opposite_angle Boolean Required Reverse angle

Response

{
    "success": true
}
